# 个人网站 IP/短视频编导定位更新设计

日期：2026-07-17  
状态：已口头批准方案 B，待主人复核书面设计

## 1. 目标

将 `hehongxiang.com` 的主叙事从“AI 剪辑师”统一为“IP/短视频编导”，同时保留 AI 剪辑、AI 工作流、剪辑 SOP 与数据复盘作为核心差异化能力；用主人提供的新简历替换旧简历，最终形成可本地验收的上线候选版。

本轮不推送 GitHub，不触发阿里云 OSS 部署。

## 2. 更新范围

### 2.1 简历

- 用 `何宏祥-IP编导简历-15758776675.pdf` 替换网站现有 `assets/何宏祥的简历.pdf`。
- 保持 HTML 下载路径和下载文件名不变，避免产生断链。
- 不改写或重新排版用户提供的简历内容。

### 2.2 全站定位

- SEO 标题、描述与页面首屏统一使用“IP/短视频编导”定位。
- Hero 主标签突出“IP/短视频编导”，副标签表达“AI 工作流驱动的内容生产”。
- 关于区从单一剪辑执行改为选题、策划、脚本、拍摄、剪辑、复盘的内容全链路能力。
- AI 能力区继续保留，但定位为提效与质量控制方法，而非目标岗位本身。
- 联系区与简历下载文案与新版定位一致。

### 2.3 作品与视觉

- 保留现有未提交草稿中的代表作品卡片、作品数据和 `assets/portfolio/` 图片。
- 保留建筑极简、几何科技、冷蓝配色和现有交互强度。
- 不做全面重构，不新增与求职转化无关的功能。

## 3. 内容结构

页面叙事顺序保持为：

1. Hero：我是谁、应聘什么岗位、看什么证明。
2. 数据概览：快速建立结果可信度。
3. 关于我：内容全链路与执行边界。
4. 代表作品：真实作品与结果数据。
5. 核心能力：编导、剪辑、AI、复盘与标准化。
6. 项目经历与证书：补足职业证据。
7. 方法论与知识体系：说明如何稳定复现结果。
8. 联系与简历下载：降低招聘方下一步行动成本。

## 4. 文案原则

- 主身份：IP/短视频编导。
- 能力标签：内容策划、脚本、拍摄协同、后期剪辑、数据复盘、AI 工作流。
- 结果证据：使用已确认真实且可公开的播放量、点赞量和项目数据。
- 避免把团队整体成果写成个人独立成果；保留“参与、协助、在指导下”等准确边界。
- 不使用空泛的“精通、顶尖、全能”等无法证明的宣传词。

## 5. 实现边界

- 主要修改 `index.html`，必要时对 `style.css` 与 `main.js` 做最小修正。
- 替换 `assets/何宏祥的简历.pdf`。
- 更新 `进度记录.md`，消除旧的“地质勘探简历”待办和过期定位。
- 不删除主人已有的未提交改动。
- 不提交或推送网站功能代码，除非主人后续明确批准；书面设计记录可单独本地提交。

## 6. 验收标准

- 新简历下载链接可用，下载后文件为已验收的 2 页 IP 编导简历。
- 页面标题、首屏、关于、能力与联系区不存在“地质勘探”或冲突职位定位。
- “AI 剪辑师”不再作为唯一主身份，AI 仍作为明确能力标签出现。
- 所有本地图片、脚本、样式、锚点和简历路径存在且可访问。
- 外链使用安全的 `rel="noopener"`，页面无明显空链接或误导按钮。
- 桌面端和移动端无横向溢出、文字遮挡、图片破损或不可点击元素。
- `prefers-reduced-motion` 下页面仍可正常阅读和操作。
- 基础秘密扫描无密钥、令牌或凭据泄漏。
- 完成验证报告后，才把候选版交给主人决定是否发布。

## 7. 发布与回滚

- 本轮只生成本地候选版，不执行 `git push`。
- 若后续批准发布，先展示最终差异与验证结果，再提交并推送。
- 回滚可通过 Git 恢复已跟踪文件；新简历替换前保留原文件的 Git 版本，不做不可恢复删除。

## 8. 已知限制

- 新简历包含手机号与邮箱；主人已主动提供该文件用于公开个人网站，视为允许随网站下载。
- 当前网站是原生 HTML/CSS/JS 静态站，没有现成测试框架；将使用静态检查、资源检查和浏览器视觉验收代替框架型单元测试。
- 作品数据的真实性与公开性已由主人确认，本轮不再联网核验第三方平台数据。
